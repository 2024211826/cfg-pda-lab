# 测试计划

本文档说明准备如何测试程序。这里只写测试目标、测试用例设计、输入和预期结果，不记录实际调试过程，不放运行截图。

## 1. 测试目标

测试需要覆盖两个实验功能：

- CFG 简化：消除 `epsilon` 产生式、单产生式、无用符号。
- PDA 到 CFG：构造等价 CFG，并继续调用 CFG 简化流程。

测试还需要覆盖输入输出格式、边界情况和错误输入处理。

## 2. 测试范围

### 2.1 功能测试

- CFG 正常输入。
- PDA 正常输入。
- PDA 转 CFG 后继续简化。

### 2.2 边界测试

- 空串相关情况。
- 单产生式链。
- 不可达符号。
- 不可终止符号。
- PDA 空输入转移。
- PDA 空压栈串。
- PDA 压入多个栈符号。

### 2.3 异常测试

- 未声明符号。
- 缺少开始符号。
- 产生式格式错误。
- PDA 转移格式错误。
- 模式编号错误。

## 3. CFG 测试用例

### TC-CFG-01 老师给定 CFG 样例

目的：验证完整 CFG 简化流程。

输入：

```text
N: S A B C D
T: a b c d
S: S
P:
S -> a | b A | B | c c D
A -> a b B | #
B -> a A
C -> d d C
D -> d d d
END
```

预期：

- `A -> #` 被消除。
- `S -> B` 这类单产生式被消除。
- `C` 不可从开始符号有效参与最终语言，应被删除。
- 输出文法不含非必要 `epsilon` 产生式、单产生式和无用符号。

### TC-CFG-02 无 epsilon 产生式

目的：确认无 `epsilon` 文法不会被错误修改。

输入：

```text
N: S A
T: a b
S: S
P:
S -> a A
A -> b
END
```

预期：

- 输出语言保持不变。
- 不引入新的 `epsilon` 产生式。

### TC-CFG-03 开始符号可推出空串

目的：验证开始符号可空时新增开始符号的处理。

输入：

```text
N: S A
T: a
S: S
P:
S -> A
A -> a | #
END
```

预期：

- 输出保留空串能力。
- 新开始符号不与已有符号冲突。
- 非开始符号的普通 `epsilon` 产生式被消除。

### TC-CFG-04 单产生式链

目的：验证多步单产生式闭包。

输入：

```text
N: S A B C
T: a b
S: S
P:
S -> A
A -> B
B -> C
C -> a | b
END
```

预期：

- 单产生式链会被展开，开始符号最终可以直接推出 `a` 或 `b`。
- 输出中不保留 `S -> A`、`A -> B`、`B -> C`。
- 最后删除不可达符号后，只保留从开始符号可达的部分。

### TC-CFG-05 不可达符号

目的：验证不可达符号删除。

输入：

```text
N: S A C
T: a c
S: S
P:
S -> A
A -> a
C -> c
END
```

预期：

- `C` 和 `C -> c` 被删除。

### TC-CFG-06 不可终止符号

目的：验证不可终止符号删除。

输入：

```text
N: S A B
T: a
S: S
P:
S -> A | a
A -> B
B -> A
END
```

预期：

- `A`、`B` 不能推出终结符串，应被删除。
- 保留 `S -> a`。

## 4. PDA 测试用例

### TC-PDA-01 老师给定 PDA 样例

目的：验证 PDA 到 CFG 转换和后续 CFG 简化。

输入：

```text
Q: q0 q1
T: a b
G: B z0
q0: q0
z0: z0
F:
D:
q0 b z0 -> q0 B z0
q0 b B -> q0 B B
q0 a B -> q1 #
q1 a B -> q1 #
q1 # B -> q1 #
q1 # z0 -> q1 #
END
```

预期：

- 能构造包含 `[q,z,q]` 形式非终结符的 CFG。
- 能处理空输入转移和空压栈串。
- 简化后输出不含明显无用符号。

### TC-PDA-02 只含直接弹栈转移

目的：验证压栈串为空时的产生式构造。

输入：

```text
Q: q0
T: a
G: z0
q0: q0
z0: z0
F:
D:
q0 a z0 -> q0 #
END
```

预期：

- 生成 `[q0,z0,q0] -> a`。
- 开始符号可推出 `a`。

### TC-PDA-03 含空输入弹栈转移

目的：验证 PDA 空输入对应 CFG 空产生式。

输入：

```text
Q: q0
T: a
G: z0
q0: q0
z0: z0
F:
D:
q0 # z0 -> q0 #
END
```

预期：

- 生成的 CFG 能产生空串。
- 简化时正确保留开始符号的空串能力。

### TC-PDA-04 压入三个栈符号

目的：验证实现没有把压栈长度写死为 0 或 2。

输入：

```text
Q: q0 q1
T: a
G: A B C z0
q0: q0
z0: z0
F:
D:
q0 a z0 -> q1 A B C
END
```

预期：

- 转换过程不越界、不崩溃。
- 能枚举中间状态序列并生成对应产生式。

## 5. 异常测试用例

### TC-ERR-01 CFG 未声明符号

输入：

```text
N: S
T: a
S: S
P:
S -> a A
END
```

预期：

- 程序报告 `A` 未声明。
- 不继续输出误导性的简化结果。

### TC-ERR-02 PDA 未声明状态

输入：

```text
Q: q0
T: a
G: z0
q0: q0
z0: z0
F:
D:
q0 a z0 -> q1 #
END
```

预期：

- 程序报告 `q1` 未声明。

### TC-ERR-03 转移格式错误

输入：

```text
Q: q0
T: a
G: z0
q0: q0
z0: z0
F:
D:
q0 a z0 q0 #
END
```

预期：

- 程序报告缺少 `->` 或转移格式错误。
