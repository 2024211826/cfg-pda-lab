# 需求清单

本文档用于明确本次实验“要做什么”。这里只拆解作业要求、功能需求、输入输出要求、提交物要求和验收标准，不展开算法细节，不分析学长代码，也不记录测试过程。

## 1. 作业来源

作业主题：

```text
理论计算机科学基础
上下文无关文法与下推自动机
```

本次实验包含两个部分：

1. 设计上下文无关文法的变换算法。
2. 构造与下推自动机等价的上下文无关文法。

提交时间要求：2026 年 7 月 3 日前，以组为单位提交到云平台，只需组长提交。

## 2. 功能需求

### 2.1 CFG 变换功能

程序应支持输入一个上下文无关文法 `G = (N, T, P, S)`。

程序应依次完成：

- 消除 `epsilon` 产生式。
- 消除单产生式。
- 消除无用符号。

程序应输出与原文法等价的简化文法。若原文法能生成空串，需要按设计约定保留空串语言能力。

### 2.2 PDA 到 CFG 转换功能

程序应支持输入一个下推自动机：

```text
M = (Q, T, Gamma, delta, q0, z0, Phi)
```

程序应构造与该 PDA 等价的上下文无关文法。

程序应输出构造得到的 CFG。

### 2.3 联合处理功能

程序应将 PDA 转换得到的 CFG 继续作为 CFG 变换算法的输入，输出等价的简化 CFG。

联合处理流程为：

```text
输入 PDA
构造等价 CFG
输出等价 CFG
简化该 CFG
输出简化 CFG
```

## 3. 输入需求

### 3.1 CFG 输入

CFG 输入至少需要包含：

- 非终结符集合 `N`。
- 终结符集合 `T`。
- 开始符号 `S`。
- 产生式集合 `P`。

空串输入输出统一使用 `#` 表示。

推荐支持老师给出的单字符文法样例：

```text
S -> a | b A | B | c c D
A -> a b B | #
B -> a A
C -> d d C
D -> d d d
```

### 3.2 PDA 输入

PDA 输入至少需要包含：

- 状态集合 `Q`。
- 输入字母表 `T`。
- 栈符号集合 `Gamma`。
- 转移函数 `delta`。
- 初始状态 `q0`。
- 初始栈符号 `z0`。
- 终态集合 `Phi`，空栈接受时可以为空。

本实验按空栈接受处理 PDA。空输入和空压栈串均使用 `#` 表示。

需要支持老师给出的 PDA 样例：

```text
delta(q0,b,z0) = {(q0,Bz0)}
delta(q0,b,B)  = {(q0,BB)}
delta(q0,a,B)  = {(q1,#)}
delta(q1,a,B)  = {(q1,#)}
delta(q1,#,B)  = {(q1,#)}
delta(q1,#,z0) = {(q1,#)}
```

## 4. 输出需求

### 4.1 CFG 简化输出

输出应包含：

- 简化后的非终结符集合。
- 简化后的终结符集合。
- 简化后的开始符号。
- 简化后的产生式集合。

输出应清楚展示每个非终结符对应的产生式。

### 4.2 PDA 转 CFG 输出

输出应包含：

- PDA 转换得到的等价 CFG。
- 对该 CFG 简化后的 CFG。

为了便于截图和报告整理，输出格式应稳定、可读、顺序尽量固定。

## 5. 测试验证需求

至少使用作业给出的 CFG 样例验证 CFG 变换功能。

至少使用作业给出的 PDA 样例验证 PDA 到 CFG 转换功能和联合处理功能。

除老师样例外，应补充边界测试，例如：

- 无 `epsilon` 产生式的 CFG。
- 开始符号可推出空串的 CFG。
- 含单产生式链的 CFG。
- 含不可达符号的 CFG。
- 含不可终止符号的 CFG。
- PDA 中包含空输入转移。
- PDA 中压栈串为空或含多个栈符号。

实验报告至少包含：

## 8. 验收标准

程序能够正确读取合法 CFG 输入。

程序能够输出不含非必要 `epsilon` 产生式、单产生式和无用符号的 CFG。

程序能够正确读取合法 PDA 输入。

程序能够构造与 PDA 等价的 CFG。

程序能够把 PDA 转换结果继续交给 CFG 简化流程处理。

老师给定的 CFG 样例和 PDA 样例能够正常运行并产生合理结果。

输出格式清晰，便于在实验报告中展示。

源码结构清楚，核心模块职责明确。
