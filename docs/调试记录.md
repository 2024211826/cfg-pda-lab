### 2026-07-01 输入首行 BOM 导致解析失败

模块：CFG/PDA 输入解析。

问题现象：使用 PowerShell here-string 作为测试输入时，程序报错 `unexpected line before P` 或 `unexpected PDA line`，错误行显示为 `N:` 或 `Q:`，但行首存在不可见字符。

复现输入：`tests/cfg_sample.txt` 和 `tests/pda_sample.txt` 对应内容。

原因定位：输入首行可能带 UTF-8 BOM，原 `trim` 只去掉普通空白字符，没有处理 BOM，导致 `startsWith(line, "N:")` 和 `startsWith(line, "Q:")` 判断失败。

修改方案：在 `common.h` 的 `trim` 中识别并跳过 UTF-8 BOM。

验证结果：老师给定 CFG 样例和 PDA 样例均可正常解析并输出结果。

关联测试用例：`cfg_sample`、`pda_sample`。

### 2026-07-01 紧凑 CFG 头部解析错误

模块：CFG 兼容输入解析。

问题现象：紧凑格式输入第一行 `SABCD` 被解析成一个整体非终结符，导致后续产生式左部 `A` 报错 `production left side is not in N`。

复现输入：`tests/cfg_compact_sample.txt`。

原因定位：解析符号集合时，只要 `splitWhitespace` 得到非空 token，就直接把整行作为一个符号集合元素；对于无空格的单字符集合，应按字符拆分。

修改方案：仅当输入行包含空白分隔时才按 token 解析；无空白时按单字符拆分，并单独取第一项作为紧凑格式开始符号。

验证结果：紧凑格式老师样例输出与正式格式一致。

关联测试用例：`cfg_compact_sample`。

### 2026-07-01 批量测试脚本异常输出不干净

模块：测试脚本。

问题现象：异常用例虽然退出码符合预期，但输出文件中混入 PowerShell 的 `NativeCommandError` 调用栈，不适合放入报告。

复现输入：`tests/err_cfg_undeclared_symbol.txt`。

原因定位：脚本使用管道和 `2>&1` 调用原生程序时，PowerShell 会把 stderr 包装成错误记录。

修改方案：改用 `.NET Process` 手动重定向标准输入、标准输出和标准错误，只把程序原始输出写入 `tests/output/`。

验证结果：异常用例输出文件只保留程序错误信息，例如 `Error: undeclared grammar symbol: A`。

关联测试用例：`err_cfg_undeclared_symbol`、`err_pda_undeclared_state`。

### 2026-07-01 保留字和 PDA 转移格式加固

模块：CFG/PDA 输入校验。

问题现象：静态复查发现，`#`、`epsilon`、`eps` 可能被声明为普通非终结符、终结符、状态或栈符号；PDA 箭头转移右侧如果只写下一状态，例如 `q0 a z0 -> q0`，会被解释为空压栈串。

复现输入：`tests/err_cfg_reserved_terminal.txt`、`tests/err_pda_reserved_stack_symbol.txt`、`tests/err_pda_missing_push.txt`。

原因定位：原校验只检查符号是否已声明，没有把空串标记作为保留字处理；PDA 箭头转移解析只判断右侧非空，没有要求压栈串字段存在。

修改方案：在 `validateCFG` 和 `validatePDA` 中拒绝集合里声明空串标记；在 PDA 箭头转移解析中要求右侧至少包含下一状态和压栈串两个字段。

验证结果：批量测试 16 个用例全部通过，新增 3 个异常用例均按预期报错并返回 1。

关联测试用例：`err_cfg_reserved_terminal`、`err_pda_reserved_stack_symbol`、`err_pda_missing_push`。
