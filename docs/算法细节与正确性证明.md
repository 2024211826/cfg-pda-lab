# 算法细节与正确性证明

本文档专门记录实验中各个核心算法的细节、正确性说明和复杂度分析。

边界说明：

- `PPT阅读总结.md` 只整理课程知识背景，不展开项目算法证明。
- `设计说明.md` 主要描述代码结构、模块划分、数据结构和程序流程。
- 本文档负责说明算法为什么这样做、为什么正确、复杂度如何。

## 1. 消除 epsilon 产生式

### 问题定义

输入一个上下文无关文法 `G = (N, T, P, S)`，构造等价文法 `G'`，使其不含非必要的 epsilon 产生式。

若原文法可以推出空串，则允许通过新增开始符号保留空串。

### 算法步骤

1. 计算可空非终结符集合 `Nullable`。
2. 对每条产生式 `A -> X1X2...Xk`，枚举右部中属于 `Nullable` 的符号保留或删除的所有组合。
3. 删除空右部产生式。
4. 如果开始符号 `S` 属于 `Nullable`，新增开始符号 `S'`，加入 `S' -> S | epsilon`。

### 正确性思路

`Nullable` 中的每个非终结符都能推出空串。因此，对原产生式右部中可空符号的任意删除，都对应原文法中的一次合法推导：被删除的符号在原文法中推出空串。

反过来，原文法中使用 epsilon 产生式导致某些可空符号消失的推导，都能由新文法中对应的“删除组合”产生式直接模拟。

因此，新文法保留了原文法除空串处理外的所有句子。若 `S` 可空，则通过新开始符号单独保留空串。

### 复杂度

设产生式右部最大长度为 `m`，可空符号最多出现 `r` 次，则单条产生式最多生成 `2^r` 条新产生式。整体复杂度主要由组合枚举决定。

## 2. 消除单产生式

### 问题定义

单产生式指形如 `A -> B` 的产生式，其中 `A`、`B` 都是非终结符。目标是构造等价文法，消除所有单产生式。

### 算法步骤

1. 对每个非终结符 `A`，计算可通过零步或多步单产生式到达的非终结符集合 `Unit(A)`。
2. 对每个 `B in Unit(A)`，把 `B` 的所有非单产生式加入 `A` 的产生式集合。
3. 删除所有单产生式。

### 正确性思路

如果 `A` 通过单产生式链到达 `B`，且 `B -> alpha` 是非单产生式，则原文法存在推导 `A =>* B => alpha`。新文法加入 `A -> alpha`，直接模拟这段推导。

反过来，新文法新增的每条 `A -> alpha` 都来自某个原文法中的单产生式链和非单产生式，因此不会产生原文法不能产生的新句子。

### 复杂度

可用闭包计算完成。若非终结符数为 `n`，产生式数为 `p`，朴素实现复杂度约为 `O(n * p * n)`，可通过图可达性优化。

## 3. 消除无用符号

### 问题定义

无用符号包括：

- 不可终止符号：不能推出纯终结符串。
- 不可达符号：不能从开始符号推导到。

目标是删除所有包含无用符号的产生式，并更新符号集合。

### 算法步骤

1. 计算可终止非终结符集合 `Generating`。
2. 删除左部不在 `Generating` 中，或右部含不可终止非终结符的产生式。
3. 从开始符号 `S` 出发，计算可达符号集合 `Reachable`。
4. 删除不可达符号及相关产生式。

### 正确性思路

不能推出终结符串的符号不可能出现在任何最终句子中，删除它们不会减少语言中的终结符串。

从开始符号不可达的符号永远不会参与任何从 `S` 开始的推导，删除它们也不会影响语言。

保留下来的产生式只包含可终止且可达的符号，因此文法不再含无用符号。

### 复杂度

两阶段都可以用集合迭代到不动点。若产生式数为 `p`，右部总长度为 `L`，朴素实现通常为 `O(p * L)` 到 `O(n * p * L)`。

## 4. PDA 到 CFG 转换

### 问题定义

输入下推自动机 `M = (Q, Sigma, Gamma, delta, q0, z0, F)`，构造等价上下文无关文法 `G`。

根据实验给出的 PDA 接受方式，若为空栈接受，则开始产生式通常构造为：

```text
S -> [q0,z0,q]
```

其中 `q` 遍历所有可能状态。

### 非终结符含义

非终结符 `[p,A,q]` 表示：

```text
从状态 p 出发，栈顶为 A，读入某个字符串后，到达状态 q，并且 A 被弹出。
```

### 算法步骤

1. 为每个三元组 `(p, A, q)` 创建非终结符 `[p,A,q]`。
2. 创建开始符号 `S`。
3. 对所有状态 `q`，加入 `S -> [q0,z0,q]`。
4. 对 PDA 的每条转移构造 CFG 产生式。
5. 对生成的 CFG 再执行 epsilon 产生式、单产生式和无用符号消除。

### 转移构造规则

若有转移：

```text
delta(p, a, A) contains (r, epsilon)
```

则加入：

```text
[p,A,r] -> a
```

若有转移：

```text
delta(p, a, A) contains (r, B1B2...Bk)
```

则对所有中间状态序列 `q1, q2, ..., qk` 加入：

```text
[p,A,qk] -> a [r,B1,q1] [q1,B2,q2] ... [q(k-1),Bk,qk]
```

当 `a` 为空输入时，产生式右部不写终结符 `a`。

### 正确性思路

非终结符 `[p,A,q]` 的定义直接对应 PDA 中“从状态 `p` 开始处理并最终弹出栈顶 `A`，到达状态 `q`”的计算片段。

当 PDA 一步弹出 `A` 且不压入新栈符号时，对应 CFG 产生式直接生成该步消耗的输入符号。

当 PDA 弹出 `A` 并压入 `B1B2...Bk` 时，后续必须依次消除这些栈符号。CFG 通过枚举中间状态，把这段连续计算拆成若干 `[state, stackSymbol, state]` 片段。

因此，PDA 的每段合法计算都能被 CFG 推导模拟；CFG 的每条推导也对应 PDA 的合法转移序列。开始产生式覆盖所有可能结束状态，所以二者接受同一语言。

### 复杂度

非终结符数量为 `|Q|^2 * |Gamma| + 1`。

若某条转移压入 `k` 个栈符号，需要枚举 `|Q|^k` 种中间状态组合。因此 PDA 到 CFG 的产生式数量会随压栈长度指数增长。
