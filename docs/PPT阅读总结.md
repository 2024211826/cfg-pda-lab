## 1. 绪论

课程从计算机科学的理论基础出发，强调形式语言与自动机的核心作用：用严格的数学模型描述计算过程、语言结构和计算能力边界。

绪论部分给出的主线是：

- 形式语言：描述符号串集合。
- 自动机：识别符号串集合。
- 文法：生成符号串集合。
- 图灵机：刻画一般计算过程。
- 可计算性与复杂性：讨论问题能不能被计算，以及计算代价有多高。

这一部分的重点不是具体算法，而是建立抽象建模意识：把现实问题或程序语言结构转化为符号、规则、状态和计算过程。

## 2. 语言与文法

第二章介绍形式语言的基础概念。

字母表是有限符号集合，字符串是字母表上符号组成的有限序列，空串记为 `epsilon`。字符串可以进行连接、求长度、取前缀、后缀、子串等操作。字母表的星闭包表示所有有限字符串及空串，正闭包表示所有非空字符串。

语言被定义为某个字母表星闭包的子集，也就是字符串集合。语言可以是有限集合，也可以是无限集合；有限语言可以列举，无限语言通常需要文法或自动机描述。

文法是语言的产生系统，形式上写为：

```text
G = (N, T, P, S)
```

其中 `N` 是非终结符集合，`T` 是终结符集合，`P` 是产生式集合，`S` 是开始符号。文法通过从 `S` 出发不断应用产生式，生成只含终结符的句子。由文法生成的所有句子的集合就是该文法的语言。

课件还用 BNF 说明了文法作为“元语言”的作用：文法不是要列出所有句子，而是通过有限规则描述一类句子的构造方式。例如标识符、时间格式、程序语句等都可以用规则描述其合法形式。这一点和后续实验中的 CFG 输入是一致的：输入并不是语言中的所有字符串，而是产生这些字符串的规则集合。

推导是文法产生句子的过程。若有产生式 `A -> beta`，则可以在某个句型中把非终结符 `A` 替换为 `beta`。从开始符号 `S` 出发，经若干步推导得到的字符串称为句型；若该字符串只由终结符组成，则称为句子。文法产生的语言可以写为：

```text
L(G) = { w | w in T* 且 S =>* w }
```

因此，判断一个文法是否被正确变换，核心是看变换前后能够生成的终结符串集合是否保持一致。

这一章还介绍 Chomsky 文法体系：

- 0 型文法：无限制文法，对应图灵机。
- 1 型文法：上下文有关文法，对应线性有界自动机。
- 2 型文法：上下文无关文法，对应下推自动机。
- 3 型文法：正则文法，对应有限自动机。

文法分类的本质是对产生式形式施加限制。限制越强，描述能力越弱，但分析和识别通常越容易。

## 3. 有限自动机与正则语言

第三章围绕有限自动机和右线性文法展开。

DFA 是确定有限自动机，由状态集合、输入字母表、转移函数、初始状态和终态集合组成。它逐个读取输入符号，每一步的后继状态唯一。一个字符串被接受，意味着输入读完后自动机处于终态。

NFA 是非确定有限自动机。与 DFA 相比，NFA 在某个状态读入某个符号后可以有多个后继状态，也可能没有后继状态。NFA 接受字符串的条件是：存在至少一条计算路径能在输入结束后到达终态。

带 epsilon 转换的 NFA 允许不消耗输入符号而改变状态。课件引入了 epsilon 闭包，用来描述从某个状态或状态集合出发，仅通过 epsilon 边能到达的全部状态。

课件给出了有限自动机之间的等价关系：

- DFA 是 NFA 的特例。
- NFA 可以通过子集构造法转化为等价 DFA。
- epsilon-NFA 也可以通过闭包计算转化为等价的普通 NFA 或 DFA。

第三章后半部分把有限自动机、正则表达式和右线性文法联系起来。正则语言可以由有限自动机识别，也可以由右线性文法生成，还可以由正则表达式描述。这说明三种形式本质上描述的是同一类语言。

此外，课件介绍了 DFA 极小化、泵浦引理和正则语言的封闭性。泵浦引理主要用于证明某些语言不是正则语言；封闭性说明正则语言在并、连接、闭包等运算下仍保持正则。

## 4. 上下文无关文法与下推自动机

第四章是课程中与语法结构关系最紧密的一章。

上下文无关文法的产生式左部是单个非终结符。它适合描述递归结构，例如括号匹配、表达式嵌套和程序设计语言语法。有限自动机没有额外存储能力，不能识别 `a^n b^n` 这类需要计数配对的语言；下推自动机通过引入栈，能够处理这类上下文无关结构。

课件首先介绍推导、归约、推导树和文法二义性。若同一个句子存在两棵不同推导树，或存在两个不同的最左/最右推导，则文法具有二义性。二义性对语法分析有重要影响，例如表达式文法需要体现运算符优先级和结合性。

### 4.1 上下文无关文法变换

`ch4.2` 的重点是 CFG 的简化和标准化。课件把 CFG 变换分为两类：一类是消除影响分析的特殊产生式或符号，另一类是把文法整理成标准形式，如 Chomsky 范式和 Greibach 范式。本次实验主要涉及第一类：消除无用符号、消除 epsilon 产生式、消除单产生式。

无用符号包括两类：

- 非生成符号：不能推出任何终结符串的符号。
- 不可达符号：不能从开始符号 `S` 推导到的符号。

课件对“有用符号”的定义是：某个符号既要能出现在从开始符号出发的推导过程中，又要能参与最终生成终结符串。消除无用符号时通常先计算可生成符号，再计算可达符号。课件特别强调，这两个步骤不能随意颠倒；若先删不可达符号，可能留下后续才会暴露出来的多余产生式。

epsilon 产生式是右部为空串的产生式。它在设计文法时很方便，但会给文法规范化和语法分析带来额外情况。课件指出，消除 epsilon 产生式后，除了原文法能否产生空串这一点需要单独处理外，不应影响原文法中其他字符串的产生。为此需要先计算可致空符号，也就是能够推出空串的非终结符。若右部中某些符号可致空，就需要考虑这些符号保留或消失后形成的各种右部组合。

单产生式是形如 `A -> B` 的产生式，其中 `A` 和 `B` 都是非终结符。它不会直接产生终结符内容，只是把推导转交给另一个非终结符，因此会增加推导链长度，也会让文法结构显得冗余。消除单产生式时，课件引入单元偶对：如果 `A` 只通过单产生式可以推导到 `B`，就把 `B` 的非单产生式补到 `A` 上，从而跳过中间的单产生式链。

本次实验要求的 CFG 简化顺序与课件小结一致：

```text
先消除 epsilon 产生式
再消除单产生式
最后消除无用符号
```

这个顺序的意义在于：前两步可能产生新的冗余符号或无用产生式，最后统一清理无用符号更自然。课件中的总体结论是，若原 CFG 至少能产生一个非空字符串，则经过这些简化后，可以得到语言等价或仅差空串处理的简化 CFG。

课件还提到 Chomsky 范式、Greibach 范式和消除左递归。它们和本次实验不是同一个实现目标，但反映了 CFG 变换的共同目的：把自由度较高的产生式整理成更适合证明、分析或程序处理的形式。例如递归下降分析不适合左递归文法，因为在没有消耗输入的情况下递归调用自身会导致死循环。

### 4.2 下推自动机

PDA 被定义为带输入带、有限控制器和下推栈的自动机。它的动作由当前状态、当前输入符号和栈顶符号共同决定，可以替换栈顶、压栈、弹栈，也可以进行 epsilon 转换。PDA 的格局通常写为：

```text
(q, w, alpha)
```

其中 `q` 是当前状态，`w` 是待输入串，`alpha` 是当前栈内容。

PDA 的形式定义是七元组：

```text
M = (Q, T, Gamma, delta, q0, z0, F)
```

其中 `Q` 是状态集合，`T` 是输入字母表，`Gamma` 是栈符号集合，`delta` 是转移函数，`q0` 是初始状态，`z0` 是初始栈符号，`F` 是终态集合。

转移函数的典型形式是：

```text
delta(q, a, Z) contains (p, alpha)
```

它表示：当前处于状态 `q`，看到输入符号 `a`，栈顶为 `Z` 时，自动机可以转入状态 `p`，并用 `alpha` 替换栈顶 `Z`。如果 `alpha` 为空串，表示弹出栈顶符号；如果 `a` 为空串，表示不消耗输入字符，仅根据状态和栈顶进行移动。

课件用 `a^n b^n` 说明 PDA 的栈能力：读入前半段 `a` 时压栈，读入后半段 `b` 时弹栈，通过栈的后进先出特性完成数量匹配。有限自动机不能保存任意多的 `a` 的数量，而 PDA 可以把这些信息放进栈中。

PDA 有两种接受方式：

- 终态接受：输入读完后进入终态。
- 空栈接受：输入读完后栈被弹空。

课件说明这两种接受方式在语言识别能力上等价。空栈接受时，终止状态不重要，只要输入读完且栈为空即可；因此常把终态集合写为空集。本次作业给出的 PDA 写作 `Phi`，正是偏向空栈接受的表示方式。

### 4.3 CFG 与 PDA 的等价关系

第四章还给出了 CFG 与 PDA 的等价关系：

- 从 CFG 可以构造等价 PDA，PDA 用栈模拟最左推导。
- 从 PDA 可以构造等价 CFG，CFG 用非终结符描述 PDA 从某状态出发、弹出某栈符号并到达另一状态的计算片段。

这一等价关系说明：上下文无关文法是生成模型，下推自动机是识别模型，但二者刻画的是同一类语言。

从 CFG 构造 PDA 时，课件给出的思路是让 PDA 进行自顶向下分析。初始时把文法开始符号压入栈；若栈顶是非终结符，就选择某条产生式，用产生式右部替换栈顶；若栈顶是终结符，则它必须和当前输入符号相同，然后弹栈并读入下一个字符。这个过程模拟的是文法的最左推导。

从 PDA 构造 CFG 是本次实验更直接相关的方向。课件采用形如 `[q, z, r]` 的非终结符，其中 `q`、`r` 是状态，`z` 是栈符号。它的含义可以理解为：PDA 从状态 `q` 出发，栈顶为 `z`，读入某段字符串后到达状态 `r`，并且这段计算最终把 `z` 弹出。

基于这个含义，开始符号会产生若干 `[q0,z0,q]`，表示从初始状态和初始栈符号出发，可能在任意状态结束并清空初始栈符号。若某条 PDA 转移直接弹出栈顶，就能产生一条较短的 CFG 产生式；若某条转移弹出一个栈符号后又压入多个栈符号，则 CFG 需要枚举中间状态，把“依次清理这些新栈符号”的过程拆成多个非终结符片段。

课件例题中，PDA 转 CFG 后还会得到许多暂时无用的非终结符和产生式，所以需要继续消除非生成符号、不可达符号，必要时再消除 epsilon 产生式和单产生式。这也解释了作业第二部分为什么要求：先由 PDA 构造等价 CFG，再把输出的 CFG 作为输入交给 CFG 变换算法继续简化。

上下文无关语言的性质部分介绍了 CFL 泵浦引理、封闭性和判定问题。与正则语言相比，CFL 的封闭性质更弱，例如对并、连接、闭包封闭，但对交一般不封闭。

第四章最后讨论了上下文无关文法在编译器和语法分析中的应用。正则语言适合词法分析，CFG 更适合描述语句、表达式、嵌套结构等语法层面的递归结构。

## 5. 图灵机

第五章引入图灵机作为更一般的计算模型。

图灵机由有限控制器、无限带、读写头、带符号集合、输入符号集合、转移函数、开始状态、空白符和终态集合组成。与有限自动机相比，图灵机可以读写带上的符号，并且读写头可以左右移动，因此具有更强的计算能力。

图灵机的格局描述某一时刻的状态、带内容和读写头位置。图灵机接受语言的方式是：从初始格局出发，经若干步转移后进入终态。

课件通过识别 `a^n b^n`、`0^n 1^n 2^n` 等例子说明，图灵机能处理有限自动机和下推自动机无法处理的一些语言结构。图灵机也可以被看作函数计算机，例如用一进制编码整数并在带上完成计算。

构造技术部分介绍了状态存储、多道图灵机、核对符和移位等技巧。这些技巧是为了方便设计图灵机，并不增加图灵机的理论计算能力。

图灵机部分还引出停机问题：是否存在一个通用算法，能判断任意程序在任意输入上是否停机。课件指出停机问题不可判定，这是可计算性理论中的核心结论之一。

## 6. 计算复杂性与可计算性基础

附录从自动机、可计算性和复杂性三个方向总结计算理论的基本问题。

复杂性部分介绍大 O 和小 o 记号，用于描述算法执行时间或空间随输入规模增长的趋势。大 O 给出上界，小 o 表示严格低阶增长。

P 类问题是在确定型图灵机上可在多项式时间内判定的问题。NP 类问题可以理解为在非确定型图灵机上多项式时间内可判定，或其解可以在多项式时间内验证的问题。

NP-complete 问题同时满足两个条件：它属于 NP，并且所有 NP 问题都能多项式时间规约到它。NP-hard 问题至少和所有 NP 问题一样难，但不一定属于 NP。

可计算性部分区分可判定问题和不可判定问题。若存在一个对所有输入都停机并给出接受或拒绝结果的图灵机，则该语言或问题是可判定的。停机问题是典型的不可判定问题。

## 7. 课程主线整理

整套课件可以按“语言的描述方式”理解：

```text
文法负责生成语言
自动机负责识别语言
表达式负责描述语言
图灵机负责刻画一般计算
复杂性理论负责衡量计算代价
```

从语言类型看，课程主线是：

```text
正则语言
  -> 有限自动机 / 正则表达式 / 右线性文法

上下文无关语言
  -> 上下文无关文法 / 下推自动机

递归可枚举语言
  -> 图灵机
```

从能力层次看，模型能力逐步增强：

```text
有限自动机：只有有限状态，不能记住任意长历史
下推自动机：增加一个栈，能处理嵌套和配对结构
图灵机：拥有可读写的无限带，刻画通用计算
```

## 8. 易混概念

`epsilon` 与空语言不同。`epsilon` 是长度为 0 的字符串，空语言是不含任何字符串的集合。

句型与句子不同。句型可以含非终结符，句子只能由终结符组成。

DFA 与 NFA 的识别能力相同，但计算方式不同。NFA 的非确定性提高了描述方便性，不提高可识别语言类别。

右线性文法与上下文无关文法不是并列关系。右线性文法属于更受限制的一类文法，其语言是正则语言；上下文无关文法描述能力更强。

PDA 的终态接受和空栈接受是两种不同定义方式，但接受能力等价。

文法二义性是文法性质，不是单个句子的普通歧义描述。一个语言可能存在有二义文法，也可能存在无二义文法；有些语言具有固有二义性。

泵浦引理通常用于否定某语言属于某类语言，而不是直接构造识别器。

图灵机的构造技巧不会增强理论计算能力，只是让设计过程更方便。
