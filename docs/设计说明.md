# 设计说明

本文档说明本实验程序的实现设计，重点描述数据结构、模块划分、算法调用流程、主程序流程和输入输出格式。算法正确性证明与复杂度分析放在 `算法细节与正确性证明.md`，测试用例与执行记录不放在本文档中。

## 1. 设计目标

程序需要完成两个功能：

1. 输入一个上下文无关文法，依次消除 `epsilon` 产生式、单产生式和无用符号，输出等价的简化文法。
2. 输入一个下推自动机，构造与之等价的上下文无关文法；再把该文法交给第一个功能进行简化，输出简化结果。

根据 `学长代码总结.md` 中的审查结论，本项目不采用“把复杂非终结符强制重命名为单字符”的方案。程序内部统一使用字符串表示符号，以支持 `q0`、`z0`、`[q0,z0,q1]` 这类多字符对象。

## 2. 技术选型

建议使用 C++17 实现。

主要理由：

- STL 的 `set`、`map`、`vector`、`string` 适合表示文法集合、产生式集合和 PDA 转移函数。
- C++17 支持结构化绑定，便于遍历 `map` 和 `tuple`。
- 与学长参考代码和课程报告环境一致，便于编译和提交。

## 3. 统一约定

### 3.1 空串表示

理论说明中的 `epsilon` 表示空串。程序输入输出中统一使用 `#` 表示 `epsilon`。

在 CFG 中：

```text
A -> #
```

表示 `A -> epsilon`。

在 PDA 中：

```text
q1 # B -> q1 #
```

左侧输入符号 `#` 表示空输入，右侧压栈串 `#` 表示弹出栈顶后不再压入任何符号。

程序内部不应把 `#` 当成普通终结符保存。CFG 的空右部用空向量表示，PDA 的空输入和空压栈串也用空字符串或空向量表示。

### 3.2 PDA 接受方式

本实验按空栈接受方式处理 PDA。也就是说，输入读完且栈被弹空时即认为接受，最终停在哪个状态不影响接受结果。

因此 PDA 的终态集合可以为空。输入格式中仍保留 `F:` 字段，是为了保持七元组形式完整；当使用空栈接受时，`F:` 可以不填。

PDA 到 CFG 转换时，开始产生式按空栈接受构造：

```text
S -> [q0,z0,q]
```

其中 `q` 遍历所有状态。

## 4. 源码结构

当前源码按实验规模合并为 5 个主要文件，解析、化简和转换逻辑分别集中在 `cfg.cpp` 与 `pda.cpp` 中：

```text
src/
  main.cpp      命令行入口、模式选择、主流程控制
  common.h      字符串裁剪、分割、连接和 epsilon 标记判断
  cfg.h/cpp     CFG 数据结构、输入解析、校验、化简和输出
  pda.h/cpp     PDA 数据结构、输入解析、校验和 PDA 到 CFG 转换
```

这种组织方式避免了课程实验中过度拆分文件，同时保留了清晰的函数边界，例如 `parseCFG`、`simplifyCFG`、`parsePDA` 和 `convertPDAToCFG`。

## 5. 核心数据结构

### 5.1 符号表示

所有终结符、非终结符、状态和栈符号都使用 `std::string` 表示。

```cpp
using Symbol = std::string;
using State = std::string;
using StackSymbol = std::string;
```

这样可以直接表示：

```text
S
A
q0
z0
[q0,z0,q1]
```

不要使用 `char` 作为核心符号类型，否则 PDA 到 CFG 后得到的复杂非终结符会难以稳定表示。

### 5.2 产生式

产生式右部使用符号序列表示：

```cpp
using RHS = std::vector<Symbol>;
```

若 `RHS` 为空，表示 `epsilon`。

一个 CFG 可以设计为：

```cpp
struct CFG {
    std::set<Symbol> nonterminals;
    std::set<Symbol> terminals;
    Symbol start;
    std::map<Symbol, std::set<RHS>> productions;
};
```

其中 `productions[A]` 表示所有左部为 `A` 的右部集合。`std::set<RHS>` 可自动去重，输出时也更稳定。

### 5.3 PDA

PDA 转移函数以“当前状态、输入符号、栈顶符号”为键。

```cpp
struct TransitionKey {
    State state;
    Symbol input;
    StackSymbol stackTop;
};

struct TransitionValue {
    State nextState;
    std::vector<StackSymbol> pushSymbols;
};

struct PDA {
    std::set<State> states;
    std::set<Symbol> inputSymbols;
    std::set<StackSymbol> stackSymbols;
    std::map<TransitionKey, std::set<TransitionValue>> delta;
    State startState;
    StackSymbol startStack;
    std::set<State> finalStates;
};
```

若输入符号为 `#`，内部记为空输入；若压栈串为 `#`，内部记为空向量，表示弹出栈顶后不再压入新符号。

## 6. 模块设计

### 6.1 CFG 解析模块

职责：

- 读取非终结符集合。
- 读取终结符集合。
- 读取开始符号。
- 读取产生式集合。
- 将输入中的 `#` 转为内部空右部。

为避免多字符符号歧义，正式输入格式采用空格分隔符号。右部中的每个符号也用空格分隔。

### 6.2 CFG 简化模块

对外提供一个总入口：

```cpp
CFG simplifyCFG(const CFG& cfg);
```

内部按顺序调用：

```text
removeEpsilonProductions
removeUnitProductions
removeUselessSymbols
```

各函数只返回新文法，不直接修改输入文法，便于调试和阶段性输出。

### 6.3 PDA 解析模块

职责：

- 读取状态集合、输入字母表、栈符号集合。
- 读取初始状态和初始栈符号。
- 读取终态集合。若为空栈接受，可允许终态集合为空。
- 读取转移函数。

由于栈符号可能是多字符，例如 `z0`，压栈串建议使用空格分隔：

```text
q0 b z0 -> q0 B z0
q1 # z0 -> q1 #
```

若需要兼容作业中的紧凑写法 `Bz0`，解析器可以在已知栈符号集合的基础上进行最长匹配切分。但正式推荐格式仍使用空格分隔，避免歧义。

### 6.4 PDA 到 CFG 模块

对外提供：

```cpp
CFG convertPDAToCFG(const PDA& pda);
```

转换时直接生成字符串形式的复杂非终结符，例如：

```text
[q0,z0,q1]
```

不再额外执行 `transfer()` 式单字符重命名。

### 6.5 输出模块

输出模块负责把内部结构格式化为可读文本。

CFG 输出顺序建议固定为：

```text
N = {...}
T = {...}
S = ...
P:
  A -> ...
  B -> ...
```

若右部为空，输出 `#`。

## 7. CFG 简化流程设计

### 7.1 消除 epsilon 产生式

输入：任意 CFG。

输出：不含非必要 `epsilon` 产生式的 CFG。

流程：

1. 计算可空非终结符集合 `Nullable`。
2. 遍历每条产生式，枚举右部中可空符号的保留和删除组合。
3. 删除普通非终结符的空右部产生式。
4. 若原开始符号可空，则创建新的开始符号，例如 `S0`，并加入 `S0 -> S | #`。

新开始符不能硬编码为某个固定字符，应通过函数生成：

```text
S0, S1, S2, ...
```

直到不与已有终结符或非终结符冲突。

### 7.2 消除单产生式

输入：已处理 `epsilon` 产生式的 CFG。

输出：不含形如 `A -> B` 的单产生式的 CFG。

流程：

1. 对每个非终结符 `A`，计算单产生式可达集合 `Unit(A)`。
2. 对每个 `B in Unit(A)`，把 `B` 的所有非单产生式加入 `A`。
3. 不再保留原单产生式。

判断单产生式时，要求右部长度为 1，且该唯一符号属于非终结符集合。

### 7.3 消除无用符号

输入：已消除 `epsilon` 和单产生式的 CFG。

输出：不含不可终止和不可达符号的 CFG。

流程：

1. 计算可终止非终结符集合 `Generating`。
2. 删除不能终止的非终结符及相关产生式。
3. 从开始符号出发计算可达符号集合 `Reachable`。
4. 删除不可达非终结符、终结符及相关产生式。

集合计算采用“不动点迭代”：每轮只根据旧集合推导新集合，直到集合大小和内容不再变化。

## 8. PDA 到 CFG 流程设计

### 8.1 非终结符构造

对于每个 `p in Q`、`A in Gamma`、`q in Q`，构造非终结符：

```text
[p,A,q]
```

其含义是：PDA 从状态 `p` 出发，栈顶为 `A`，读入某段字符串后到达状态 `q`，并且这一段计算最终弹出 `A`。

### 8.2 开始产生式

创建新开始符号 `S`。对每个 `q in Q`，加入：

```text
S -> [q0,z0,q]
```

这样可以覆盖空栈接受时可能停在任意状态的情况。

### 8.3 转移产生式

若有 PDA 转移：

```text
delta(p, a, A) contains (r, #)
```

即弹出栈顶后不压入新符号，则加入：

```text
[p,A,r] -> a
```

若 `a` 为空输入，则右部为空，即 `epsilon` 产生式。

若有 PDA 转移：

```text
delta(p, a, A) contains (r, B1 B2 ... Bk)
```

则枚举所有中间状态序列 `q1, q2, ..., qk`，加入：

```text
[p,A,qk] -> a [r,B1,q1] [q1,B2,q2] ... [q(k-1),Bk,qk]
```

其中 `a` 为空输入时不输出终结符 `a`。

该设计支持任意压栈长度 `k`。当 `k = 0` 时使用直接弹栈规则；当 `k >= 1` 时通过递归或循环生成所有状态序列。

### 8.4 转换后简化

PDA 转 CFG 会生成大量未必有用的非终结符和产生式。因此第二个实验的完整流程为：

```text
读取 PDA
构造等价 CFG
输出未化简 CFG
调用 CFG 简化流程
输出简化 CFG
```

## 9. 主程序流程

程序启动后提供两种模式：

```text
1. Simplify CFG
2. Convert PDA to CFG and simplify
```

模式 1：

```text
读取 CFG
检查基本合法性
调用 simplifyCFG
输出简化后的 CFG
```

模式 2：

```text
读取 PDA
检查基本合法性
调用 convertPDAToCFG
输出等价 CFG
调用 simplifyCFG
输出简化后的 CFG
```

基本合法性检查包括：

- 开始符号必须属于非终结符集合。
- 产生式左部必须属于非终结符集合。
- 产生式右部符号必须属于非终结符或终结符集合。
- PDA 初始状态必须属于状态集合。
- PDA 初始栈符号必须属于栈符号集合。
- PDA 转移中的状态、输入符号和栈符号必须属于对应集合，`#` 除外。

## 10. 输入格式设计

### 10.1 CFG 输入格式

推荐格式：

```text
N: S A B C D
T: a b c d
S: S
P:
S -> a | b A | B | c c D
A -> a b B | #
B -> a A
C -> d d C
D -> d d d
END
```

说明：

- `N:` 后列出非终结符。
- `T:` 后列出终结符。
- `S:` 后给出开始符号。
- `P:` 到 `END` 之间为产生式。
- `|` 分隔同一左部的多个右部。
- 每个右部内部用空格分隔符号。
- `#` 表示空串。

为了方便录入老师给出的紧凑样例，也可以兼容如下格式：

```text
SABCD
abcd
S a|bA|B|ccD
A abB|#
B aA
C ddC
D ddd
@
```

兼容格式只建议用于单字符终结符和非终结符。

### 10.2 PDA 输入格式

推荐格式：

```text
Q: q0 q1
T: a b
G: B z0
q0: q0
z0: z0
F:
D:
q0 b z0 -> q0 B z0
q0 b B -> q0 B B
q0 a B -> q1 #
q1 a B -> q1 #
q1 # B -> q1 #
q1 # z0 -> q1 #
END
```

说明：

- `Q:` 是状态集合。
- `T:` 是输入字母表。
- `G:` 是栈符号集合。
- `q0:` 是初始状态。
- `z0:` 是初始栈符号。
- `F:` 是终态集合。空栈接受时可为空。
- `D:` 到 `END` 之间为转移函数。
- `#` 在输入位置表示空输入，在压栈位置表示不压入任何符号。

## 11. 输出格式设计

模式 1 输出：

```text
Simplified CFG
N = {A, B, D, S}
T = {a, b, c, d}
S = S
P:
  S -> a | a A | b | b A | c c D
  A -> a b B
  B -> a | a A
  D -> d d d
```

模式 2 输出两段：

```text
Equivalent CFG generated from PDA
...

Simplified CFG
...
```

输出时保持集合和产生式稳定排序，便于人工检查和截图。

## 12. 错误处理设计

输入异常时不直接崩溃，输出明确错误信息。

常见错误包括：

- 未声明的符号出现在产生式右部。
- 产生式缺少 `->`。
- PDA 转移格式不完整。
- 压栈串中存在未声明栈符号。
- 开始符号或初始状态不存在。
- 模式编号不合法。

错误信息应尽量包含出错符号或出错内容，方便定位。

## 14. 设计注意事项

不要把复杂非终结符重命名为单字符。PDA 到 CFG 产生的 `[q,z,r]` 应直接作为合法非终结符保存。

不要硬编码新开始符。需要新增开始符时，应自动生成不冲突的名称。

不要在遍历容器时用不安全方式删除元素。若要删除，优先构造新集合；或使用 `while` 循环手动控制迭代器。

不要把 `#` 当成普通终结符。它只表示空串或空输入。

不要假设压栈串长度只能是 0 或 2。PDA 到 CFG 转换应按一般 `k` 个压栈符号处理。
